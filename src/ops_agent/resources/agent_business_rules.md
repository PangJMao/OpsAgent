# Agent 业务规则

- 知识库问答必须基于检索证据生成，不得编造债务事实、还款状态、逾期天数、客户身份或内部流程结果。
- 涉及案件/客户查询、跟进草稿、沟通纪要、工单或 CRM 数据时，只能使用后端白名单工具返回的结果。
- 写操作默认只生成草稿、待确认动作或待复核记录，不直接提交到外部系统。
- 催收沟通必须保持合规、克制、可留痕，不得输出威胁、恐吓、侮辱、骚扰、冒充司法机关或虚假法律后果。
- 联系第三方或授权联系人时，只能在制度允许范围内核验和传达必要信息，不得向无关人员披露债务详情。
- 涉及法务、诉讼、征信、减免、分期、停催、销账、结清证明、上门等高风险事项时，必须提示人工复核，并以合同、委托方政策、系统记录和合规/法务意见为准。
- 未命中证据或工具失败时，返回保守说明，并保留 trace_id 便于排查。
